# 要件定義書 — 公園周遊型クイズWebアプリ(仮称)

- 版: v1.0(2026-07-12)
- 対象: 清見台中央公園(千葉県木更津市)
- 位置づけ: 授業「問題解決技法」における公園活性化提案。最終ゴールは市役所担当者への発表で、クイズ体験の雰囲気を実際に感じてもらうこと。**動くデモを最優先**とする。

## 1. 背景・目的

- 清見台中央公園への滞在時間を延ばす(目標: +5〜10分)ため、スマホブラウザで楽しめるクイズラリー体験を提供する。
- クイズを通じて木更津市内の他の観光スポット・施設への誘導も行う。
- 将来的にアクアコイン(木更津市の電子地域通貨)と連携したポイ活機能を見据える。

## 2. ターゲットユーザー

- ファミリー層(親子連れ)、散歩で訪れる地域住民。
- アプリのインストールを好まないライトユーザー。→ **インストール不要、QRを読むだけで始まるWebアプリ**とする。

## 3. ゲームルール(確定)

- 園内4〜5箇所のスポットにQRコードを設置する。候補: カバの置物 / 集会所 / 貯水タンク滑り台 / 時計台。
- **巡回は自由順**。どのQRからでも始められる。
- 1QRにつき4択クイズを2〜3問出題。解答すると正誤判定と解説を表示し、そのスポットのスタンプを獲得。
- 時計台以外のスポットをすべてクリアすると、最終スポット「時計台」が解放される。
- 時計台のQRを読むと、コンプリート演出として**画面上で時計台(のハンドル)を回す演出**が再生される(Phase 1)。将来的に物理ハンドル連携へ拡張(Phase 2)。
- クイズ内容は市の紹介と、他の観光施設への誘導を兼ねる(クイズ例は既存の下書き10問を参照)。

## 4. 機能要件

| ID | 機能 | 内容 |
|----|------|------|
| FR-1 | スポット別URL | QRコードは `…/quiz?spot_id=N` 形式のURLを指し、対応するスポットのクイズを読み込む |
| FR-2 | クイズ解答 | 4択クイズの表示・タップ解答・正誤判定・解説表示 |
| FR-3 | 進捗管理 | 獲得スタンプ数と未発見スポット一覧の表示。進捗はlocalStorageに保存 |
| FR-4 | 時計台解放 | 時計台以外の全スポットクリアを判定し、時計台クイズ+回転演出を解放 |
| FR-5 | 位置情報チェック(ゆるめ) | 位置情報を**任意で**取得し、公園から明らかに遠い場合のみ注意表示。拒否されてもプレイ可能。判定は端末内のみで完結し、位置情報を外部に送信しない |
| FR-6 | リセット | 「はじめからやり直す」機能(進捗クリア)。デモ発表時にも必須 |
| FR-7 | クイズデータ管理 | 問題・選択肢・解説・スポット定義をJSONファイルで管理し、コードと分離する |

## 5. 非機能要件

- **ユーザビリティ**: スマホファースト。屋外の直射日光下でも読める高コントラスト配色。片手・タップのみで完結。子供にも読める平易な文言・ふりがな配慮。
- **パフォーマンス**: 屋外のモバイル回線で待たされない軽量な初回読み込み。
- **セキュリティ・プライバシー**: 個人情報は一切収集しない。進捗は端末内(localStorage)のみ。位置情報は端末内判定のみで送信・保存しない。
- **対応環境**: iOS Safari / Android Chrome の最新版を主対象。
- **可用性**: 静的サイトのためサーバー障害要因が少ない。

## 6. 技術構成

- フロントエンド: **Vite + React + TypeScript**(SPA、静的サイト)
- データ: クイズ・スポット定義はJSONファイル
- 進捗保存: localStorage
- ホスティング: GitHub Pages / Vercel 等の静的ホスティング(無料枠で運用可)
- バックエンド: Phase 1では**なし**

## 7. フェーズ計画

| フェーズ | 内容 | 備考 |
|---------|------|------|
| **Phase 1(授業発表)** | クイズラリー本体+画面上の時計台演出。デモモード(発表会場でQR紙を掲示 or 画面から直接スポット選択) | 本要件定義書のスコープ |
| Phase 2 | 時計台の物理ハンドル連携(マイコン+センサー等) | 電源・防水・保守・市との調整が必要 |
| Phase 3 | バックエンド追加+アクアコイン連携(ポイ活) | クリア判定のサーバー検証(ワンタイムコード等)が必須。localStorageのみの判定ではポイント付与の不正を防げないため |

## 8. 未決事項

- スポットの最終確定(4箇所か5箇所か)
- クイズ問題の確定(現在下書き10問。空欄の問題あり)
- 発表時のデモの見せ方(会場にQRコードを印刷して掲示するか、スクリーンで流れを見せるか)
- 実運用に進む場合の市・公園管理者との調整(QR設置許可、掲示物の耐候性)
